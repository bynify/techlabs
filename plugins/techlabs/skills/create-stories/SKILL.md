# create-stories

Create user stories with acceptance criteria.

## When to Use
- Sprint planning
- Feature specification
- Backlog management

## Execution

### Step 1: Story Format
```markdown
## Story: [Title]

**As a** [user type]
**I want to** [action]
**So that** [benefit]

### Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

### Technical Notes
- Dependencies: [list]
- Complexity: [low/medium/high]
- Estimation: [story points]
```

### Step 2: INVEST Check
- **I**ndependent: No dependencies?
- **N**egotiable: Details can be discussed?
- **V**aluable: Delivers user value?
- **E**stimable: Team can estimate?
- **S**mall: Fits in one sprint?
- **T**estable: Can verify completion?

## Output
- User stories
- Acceptance criteria
- Dependency map
- Sprint backlog
