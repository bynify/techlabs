# auto-generate-tests

Auto-generate tests from code.

## Execution

### Step 1: Analyze Code
```
SCAN:
- Function signatures
- Input/output types
- Dependencies
- Edge cases
```

### Step 2: Generate Tests
```
FOR EACH function:
1. Create test file
2. Generate table-driven tests
3. Add edge cases
4. Create mocks for dependencies
```

### Step 3: Validate
```
CHECK:
- Tests compile
- Tests run
- Coverage increase
- No false positives
```

### Step 4: Report
```
SHOW:
- Tests generated
- Coverage before/after
- Files modified
```

## Output
- Test files
- Coverage report
