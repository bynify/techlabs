# create-mvp-scope

Define MVP scope with must-have features and timeline.

## When to Use
- Project kickoff
- Feature prioritization
- Launch planning

## Execution

### Step 1: List Features
```
ALL FEATURES:
1. User authentication
2. Dashboard with analytics
3. Payment integration
4. Admin panel
5. Mobile responsive
6. Notifications
7. Export to CSV
```

### Step 2: MoSCoW Prioritization
```
MUST HAVE (MVP):
- [ ] User authentication (signup/login)
- [ ] Core feature (the main thing)
- [ ] Basic dashboard

SHOULD HAVE:
- [ ] Mobile responsive
- [ ] Basic notifications

COULD HAVE:
- [ ] Admin panel
- [ ] Export

WON'T HAVE (v2):
- [ ] Payment integration
- [ ] Advanced analytics
```

### Step 3: Estimate
```
MVP ESTIMATE:
- Auth: 5 days
- Core: 10 days
- Dashboard: 5 days
- Responsive: 3 days

TOTAL: 23 days (4.5 weeks)
BUFFER: +25% = 29 days (6 weeks)
```

### Step 4: Success Criteria
```
MVP SUCCESS:
- [ ] 50 beta users onboarded
- [ ] Core workflow completed by 80% of users
- [ ] No critical bugs
- [ ] Basic feedback collected
```

## Output
- Feature prioritization
- Timeline estimate
- Success criteria
- Scope boundaries
