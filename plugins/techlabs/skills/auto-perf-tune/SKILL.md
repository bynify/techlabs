# auto-perf-tune

Automatic performance tuning.

## When to Use
- Performance tuning
- Bottleneck resolution
- Query optimization
- Caching strategy

## Execution

### Step 1: Profile Application
```
PROFILE:
- CPU usage
- Memory usage
- Database queries
- Network calls
- I/O operations
```

### Step 2: Identify Bottlenecks
```
DETECT:
- Slow functions
- N+1 queries
- Memory leaks
- Blocking operations
- Inefficient algorithms
```

### Step 3: Optimize
```
OPTIMIZE:
- Database queries
- Caching strategy
- Algorithm efficiency
- Memory management
- Async operations
```

### Step 4: Verify
```
VERIFY:
- Performance improved
- No regression
- Tests pass
```

## Output
- Performance report
- Optimizations applied
- Benchmarks
