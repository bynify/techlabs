# product-roadmap

Strategic product roadmap with phases, milestones, and prioritization.

## When to Use
- Product planning
- Investor presentations
- Team alignment
- Quarterly planning

## Execution

### Step 1: Define Vision
```
ASK USER:
1. What's the product vision?
2. What are the key objectives?
3. Target milestones?
4. Resource constraints?
```

### Step 2: Create Theme Structure
```
QUARTERLY THEMES:

Q1: Foundation (MVP Launch)
- Core features that deliver primary value
- Technical foundation for future growth
- Launch readiness

Q2: Growth (User Acquisition)
- Features that drive adoption
- Integrations and partnerships
- Marketing and SEO

Q3: Scale (Performance & Reliability)
- Performance optimization
- Enterprise features
- International expansion

Q4: Monetization (Revenue)
- Premium features
- Pricing tiers
- Enterprise sales tools
```

### Step 3: Map Features to Quarters
```markdown
## Q1: Foundation

### Month 1: Core
- [ ] User authentication (5 pts)
- [ ] Basic dashboard (5 pts)
- [ ] Core API (13 pts)

### Month 2: Value
- [ ] Primary feature A (13 pts)
- [ ] Primary feature B (8 pts)
- [ ] Notifications (5 pts)

### Month 3: Polish
- [ ] UI/UX refinement (8 pts)
- [ ] Performance (5 pts)
- [ ] Launch prep (3 pts)

## Q2: Growth

### Month 4: Acquisition
- [ ] SEO optimization (5 pts)
- [ ] Landing pages (8 pts)
- [ ] Referral system (13 pts)

### Month 5: Engagement
- [ ] Email campaigns (8 pts)
- [ ] In-app messaging (5 pts)
- [ ] Analytics dashboard (8 pts)

### Month 6: Integration
- [ ] Slack integration (8 pts)
- [ ] API v2 (13 pts)
- [ ] Webhook system (8 pts)
```

### Step 4: Define Milestones
```
MILESTONES:

M1: MVP Launch (End of Q1)
- Core features working
- 10 beta users
- Basic monitoring

M2: Public Launch (Mid Q2)
- All P0 features
- 100 users
- Payment integration

M3: Growth (End of Q2)
- 1000 users
- $10K MRR
- Team of 5

M4: Scale (End of Q3)
- 10,000 users
- $100K MRR
- Enterprise features
```

### Step 5: Track Dependencies
```markdown
## Dependencies

| Feature | Depends On | Status |
|---------|------------|--------|
| Payment integration | Stripe account | ⏳ Waiting |
| OAuth login | Google/GitHub apps | ⏳ Waiting |
| Mobile app | API v2 | 🔴 Blocked |
| Email notifications | SendGrid setup | ✅ Ready |
```

### Step 6: Generate Roadmap Document
```markdown
# Product Roadmap 2024

## Vision
[Product vision statement]

## OKRs
### Q1
- Objective: Launch MVP
  - KR1: Ship core features by March 31
  - KR2: Onboard 10 beta users
  - KR3: Achieve <500ms API latency

### Q2
- Objective: Grow user base
  - KR1: Reach 1000 users
  - KR2: Achieve $10K MRR
  - KR3: Launch 3 integrations

## Timeline
[Visual timeline or Gantt]

## Risks
| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Stripe delays | Medium | High | Start integration early |
| Key engineer leaves | Low | High | Document everything |
| Scope creep | High | Medium | Strict prioritization |
```

## Output
- Quarterly roadmap
- Feature prioritization per quarter
- Milestone definitions
- Dependency map
- Risk assessment
