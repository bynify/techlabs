# auto-generate-docs

Auto-generate documentation from code.

## When to Use
- Documentation generation
- API docs
- README updates
- Code comments

## Execution

### Step 1: Analyze Code
```
SCAN:
- Function signatures
- API endpoints
- Database schemas
- Configuration
```

### Step 2: Generate Documentation
```
CREATE:
- API documentation (OpenAPI)
- Function documentation
- Database schema docs
- Configuration docs
```

### Step 3: Update README
```
UPDATE:
- README.md
- API docs
- Architecture docs
```

### Step 4: Validate
```
CHECK:
- Docs are accurate
- Links work
- Examples are correct
```

## Output
- Documentation files
- Updated README
