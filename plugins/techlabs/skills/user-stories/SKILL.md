# user-stories

Create user stories with acceptance criteria.

## Execution

### Step 1: Format
```markdown
## Story: [Title]

**As a** [user type]
**I want to** [action]
**So that** [benefit]

### Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3
```

### Step 2: Example
```markdown
## Story: User Login

**As a** registered user
**I want to** log in with my email
**So that** I can access my account

### Acceptance Criteria
- [ ] Valid credentials grant access
- [ ] Invalid credentials show error
- [ ] Account locks after 5 failed attempts
- [ ] Password reset link works
```

## Output
- User stories
- Acceptance criteria
- Dependencies mapped
